01 — Foundations

Motion

Purposeful, fast transitions. Never use motion for decoration. All durations are 150ms–400ms.

Hover transition transition: all 0.15s ease — Buttons, links, chips
Screen entrance opacity 0→1, translateY 16px→0, 400ms — Page sections, modals
Framer Motion initial opacity:0 y:20 → animate opacity:1 y:0 — Step transitions
Loading pulse opacity 1→0.5→1, 2s infinite — Skeleton, status indicators
Rule: Motion should reinforce the app's "high-stakes review" personality — snappy and decisive, not playful or springy.

02 — Components

Buttons

Five button variants, each with a strict semantic role. Never use a primary button for secondary actions.

Variants
Primary — One per screen. The single most important action. Zero border-radius. Red glow shadow.
Outline — Secondary actions within a section. 8px radius.
Amber — Upgrade/upsell actions only. Signals tier progression.
Purple — Multi-persona mode exclusively. Distinguishes from single analysis.
Ghost — Dismissal and low-priority actions. No border, no background.
Disabled — Same shape as primary but rgba(255,255,255,0.06) background, no glow, no cursor.
Sizes
CTA button is always full-width inside form panels (right-column layout). When used inline (hero, cards), it sizes to its content.

Rule: Layer 1 is always full-width, zero border-radius. Layer 2 is always 4px radius, color-coded to its persona type. Layer 3 uses a 2-column grid with smaller type. Layer 4 is the privacy assurance note — always present for non-premium users.

03 — Patterns

Tier Gating

Three access tiers. Visual treatment must clearly communicate the tier state without blocking the user's view.

Access Tiers
◈

Guest (Anonymous)

1 trial analysis. Upload zone active, form active, but CTA becomes "Upgrade to continue" after limit.

✓

Registered

Full analysis access. Medium length unlocked. Multi-jury shown but gated with amber "Upgrade" badge unless PROMO.

♛

Premium

All features unlocked. Long analysis, Multi-Jury, Material Analysis, AI Mentor defense mode.

Gating Treatment Rules

✓ Do

Show the locked feature with an UPGRADE badge. Keep it visible but disabled. Never hide it entirely.

✗ Don't

Don't remove locked features from the UI. Don't use modal interrupts to block the user mid-flow.

04 — UX Rules

Visual Hierarchy

One Primary Action Per Screen

Every screen must have exactly one red primary CTA button. All other actions are secondary (outline) or tertiary (ghost). Never place two red buttons in the same view.

Don't ALL-CAPS Everything

ALL-CAPS text is reserved for: navigation links, CTA button labels, cost indicators, section divider headers, and status chips. Body text, form labels, and descriptions are sentence case.

Mono = Metadata

JetBrains Mono signals "system information" — costs, IDs, statuses, progress indicators. Never use mono for descriptive body copy or instructions.

Form Panels Have Sticky CTAs

Multi-step or scrollable form panels always have a fixed footer with the primary action. The user should never need to scroll to submit. The footer is border-top + bg-1 + padding 16px 24px.

Blueprint Grid is the Stage

The grid background is a product personality element — it must always be visible on the page body. Never cover it with full-bleed solid backgrounds. Cards and panels sit on top of it.

04 — UX Rules

Feedback & Errors

Error Before Submit

Validation errors (file too large, missing field) appear inline, immediately adjacent to the problematic element. Never only in a toast. Inline errors use the red error card pattern.

Disabled State Must Explain Itself

A disabled CTA must always show a secondary label explaining why. e.g., "Face the Jury — upload a file first" or "File processing...". Never show a gray button with no context.

Toasts for Transient Confirmations Only

Toasts are for: successful payment, badge unlock, successful auto-fill. Not for errors — those belong inline. Toasts auto-dismiss after 4–7 seconds. No manual close button needed.

04 — UX Rules

Cost Indicators (Rapido)

Always Show Cost on CTAs

Every action that consumes Rapido must display the cost in the button label itself: "Face the Jury (4 Rapido)". Use the cost-badge component inline, or a parenthetical in mono/lighter weight.

Low Balance Warning

When Rapido balance falls below 15, show the amber low-balance bar above the CTA stack. Show remaining balance and a "Go Premium" link. Never use a modal for this warning.

Rapido Color is Always Red

The Rapido icon (✦), value text, and cost badges are always --red: #FF0033. This creates an instant cost-awareness association throughout the product.

04 — UX Rules

Voice & Copy

CTAs: Imperative, Present Tense

"Face the Jury" not "Submit for Analysis". "Open Studio Desk" not "Go to Studio". Direct, action-oriented. The product has a high-stakes jury personality — the language should match.

Placeholder Text: Concrete Examples

Form placeholders use "e.g. Karaköy, Istanbul" not "Enter your site location". Every placeholder should be a realistic example the user can understand immediately.

Error Messages: Cause + Fix

"File exceeds the 35 MB limit. Reduce size and re-upload." — always state what went wrong AND what to do next. Never just "Error occurred."

Section Headers: Title Case, No Verbs

"Project Context" not "Enter Project Context". "Analysis Settings" not "Configure Analysis". Headers name the content, not the action.

Quick Reference Card

  • JetBrains Mono for all UI labels
  • Inter for body and descriptions
  • Red for primary CTAs only
  • Amber for premium upsell only
  • 8px grid for all spacing

✗ Avoid

  • Inter for nav/button labels
  • ALL-CAPS for body/description text
